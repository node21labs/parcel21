    Drop-in for the RGB proxy protocol

    Parcel21 mirrors RGB HTTP JSON-RPC so wallets can adopt it as an alternative transport. Each method maps to a Nostr mechanism:

    consignment.postBuyer gift-wraps the consignment to the seller (kind 1059)
    consignment.getSeller subscribes for gift wraps addressed to it and unwraps
    ack.postSeller gift-wraps an ACK to the buyer's per-transfer reply key
    ack.getBuyer is pushed the ACK — no polling

    Encryption & transport

    Payloads are encrypted with NIP-44 and wrapped with NIP-59 gift wrap, addressed to an ephemeral key — so the relay sees an opaque blob, not the contents or the real parties. Small consignments ride inline; larger ones go to encrypted Blossom storage referenced inside the wrap.

    Security model

    Authenticated unwrap (a normative MUST). A naive gift-wrap unwrap does two decryptions and no authentication — so a forged wrap could fake an ACK and trigger a Bitcoin broadcast. Parcel21 verifies the seal signature, binds the seal's key to the rumor author, and recomputes the rumor id on every unwrap, and only broadcasts the witness tx after a verified ACK.
